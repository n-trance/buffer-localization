# Translation with i18n

This app uses **i18next** and **react-i18next** with dynamic locale loading, browser language detection, and a type-safe `useT()` hook. Translations are loaded by namespace from JSON files; keys are typed in [src/i18n/types.ts](../src/i18n/types.ts).

## Locale layout

Translations live under `public/locales/` by platform and locale. The web app loads from the `web` platform:

```
public/locales/
└── web/
    ├── en/
    │   ├── common.json
    │   ├── home.json
    │   ├── layout.json
    │   ├── journey.json
    │   └── job.json
    ├── de/
    │   ├── common.json
    │   └── ...
    └── fr/
        ├── common.json
        └── ...
```

Each JSON file is one **namespace** (e.g. `common`, `home`). Keys can be nested; the hook uses dot-notation: `namespace.section.key`.

## How it loads

```
  Browser
     |
     v
  I18nProvider (wraps app)
     |
     v
  initI18n() --> Backend loadPath: /locales/web/{{lng}}/{{ns}}.json
     |
     v
  LanguageDetector (localStorage "i18nextLng" / navigator)
     |
     v
  JSON fetched per namespace when needed
```

[I18nProvider](../src/components/I18nProvider.tsx) runs on the client and calls `initI18n()` once. The backend then fetches `/locales/web/{lng}/{ns}.json` (e.g. `/locales/web/de/common.json`) when a namespace is used. The detector uses `localStorage` first, then the browser language.

## Config

See [src/i18n/config.ts](../src/i18n/config.ts):

| Setting | Purpose |
|--------|---------|
| `supportedLngs` | Allowed locales: `en`, `de`, `fr`, `ja`, `ar` |
| `fallbackLng` | `en` when a key or locale is missing |
| `namespaces` | `common`, `home`, `layout`, `journey`, `job` |
| `defaultNS` | `common` |
| `rtlLngs` | `ar` – sets `document.documentElement.dir` and layout for RTL |
| Detection | `localStorage` key `i18nextLng`, then navigator |

## Usage in components

Use the type-safe hook from `@/i18n`:

```tsx
import { useT } from "@/i18n";

function MyComponent() {
  const t = useT();
  return (
    <>
      <h1>{t("home.intro.title")}</h1>
      <p>{t("home.intro.description", { templates: "Templates", learning: "Learning" })}</p>
    </>
  );
}
```

Keys follow the pattern `namespace.section.key`. The `home.intro.description` value in English is:

```json
"Looking for a starting point or more instructions? Head over to {{templates}} or the {{learning}} center."
```

Pass interpolation values as the second argument: `{ templates: "Templates", learning: "Learning" }`.

To switch language or read the current one, use `useTranslation()` from react-i18next:

```tsx
import { useTranslation } from "react-i18next";

function LanguageSwitcher() {
  const { t, i18n } = useTranslation();
  return (
    <select value={i18n.language} onChange={(e) => i18n.changeLanguage(e.target.value)}>
      <option value="en">{t("common.languages.en")}</option>
      <option value="de">{t("common.languages.de")}</option>
      {/* ... */}
    </select>
  );
}
```

See [src/components/LanguageSwitcher.tsx](../src/components/LanguageSwitcher.tsx) for a full example.

## Formatters

[src/i18n/formatters.ts](../src/i18n/formatters.ts) provides Intl-based helpers that use the current i18n language:

```tsx
import { formatNumber, formatDate, formatCurrency } from "@/i18n/formatters";

formatNumber(1234.56);           // "1,234.56" (en) / "1.234,56" (de)
formatDate(new Date());          // locale date string
formatCurrency(99.99, "USD");    // "$99.99" (en) / "99,99 $" (de)
```

## Adding a language

1. **Config** – In [src/i18n/config.ts](../src/i18n/config.ts), add the locale to `supportedLngs` and to `languageDisplay` (flag + native name).
2. **Types** – In [src/i18n/types.ts](../src/i18n/types.ts), add the new key under `TranslationKeys.common.languages` (e.g. `fr: null`).
3. **Translation files** – Create `public/locales/web/{lng}/` with the same namespaces as English (`common.json`, `home.json`, etc.). You can copy from `en` and translate, or use the [translate script](translate-locales.md) to generate from English.
4. **Language labels** – In every locale’s `common.json`, ensure `languages` includes the new code (e.g. `"fr": "Français"`).
5. **RTL** – If the language is RTL, add it to `rtlLngs` in config.
6. Run the app and switch to the new language to verify.

## Translations API (optional)

The route [src/app/api/v1/translations/[lng]/route.ts](../src/app/api/v1/translations/[lng]/route.ts) serves translations by `platform` and `lng` for other clients (e.g. mobile). Query params: `?platform=web&ns=common,home`. The key structure is the same as in this doc.
